异常/中断上下文

Fields§

§x: [usize; 32]

普通寄存器。 异常/中断发生的的时候不一定是系统调用，也可能是时钟中断、缺页等，所以不能认为它是一种“函数调用“。 所以上下文里要保留所有寄存器，而不只是 callee save 的

§sstatus: Sstatus

CSR 寄存器 sstatus，含中断的信息

§sepc: usize

CSR 寄存器 sepc，表示发生中断的位置

§cpu_id: usize

CPU 的编号。在内核时，这个信息存在 tp 寄存器上

§fs: [usize; 2]

浮点寄存器
